客服原话：

![](/article/sbgithub/1.png)

意思就是我开小号了。

问题来了，我在三年前就开过小号，也就是现在的大号 `yesmore`（已封），当时是从另一个号迁移过来的，为什么当时没封？

第二个问题，我大致可以猜到为什么近期才封。

在上个月三月中旬我发布了一个项目 
[yesmore/vmail](https://github.com/yesmore/vmail)（当然现在这个仓库已经404了，我迁移到
[oiov/vmail](https://github.com/oiov/vmail) 了），这个项目在社区算是小火，上了 github trending，发布10天不到
star 超过1k。


![](/article/sbgithub/2.png)

![](/article/sbgithub/3.png)

然后问题来了，在仓库 star 超过1000那天，仓库上面提示让我添加一个 `successer`，就是账户继承人，意思是在主账号那绑定另一个账号，
防止主账号寄了影响项目，相关设置在 https://github.com/settings/admin 的 `Successor settings` 那里。

Fine，我就按他说的关联了基本没用的小号，然后第二天就给我大号封了？？？

于是第一天就开始跟客服对线，从一开始委婉但是明确的告诉我不会恢复账号，我就跟他说明原因，对线一周告诉我把其他账号删了就给我恢复。

ok，我删，但是为什么不早说？

又等了三天终于回复了，这次客服换人了，告诉我没删完，直接说不会处理后续的工单了？？？

mmp，能不能直接说清楚。我又检查了一遍，有个小号都不知道啥时候的了，要不是浏览器有记录密码那个我都忘了有这个账号。

这个Github账号绑了几十个第三方APP，也登不上了。仓库还要一个个迁移，星星也没了，心累。

只能看看 [archive](https://web.archive.org/web/20240324114952/https://github.com/yesmore) 纪念一下了
